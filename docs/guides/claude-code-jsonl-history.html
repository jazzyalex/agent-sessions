Claude Code JSONL History: What You Can Recover Locally

Claude Code writes local session transcripts as JSONL. Those transcripts often contain the useful part of an old run: the prompt, assistant response, tool call, command output, error, or file path that explains what happened.

Where Claude Code Session History Lives

Standard Claude Code local history lives under:

~/.claude/projects/<encoded-cwd>/<session-id>.jsonl

Agent Sessions also scans configured Claude roots, CLAUDE_CONFIG_DIR, CLAUDE_CONFIG_DIRS, and Claude Desktop local-agent-mode session roots.

What You Can Recover

  • Old prompts and assistant responses.
  • Tool calls and tool results.
  • Command output.
  • File paths and errors mentioned in the session.
  • Old decisions or implementation notes.

The boundary is transcript context versus hidden runtime/model state. Agent Sessions can search and display transcript context that Claude wrote to disk. It should not be described as recovering hidden runtime state.

What Agent Sessions Does

  • Provides a local macOS browser for Claude Code sessions.
  • Full-text searches old transcripts.
  • Shows readable timeline views.
  • Separates Claude rows from Codex, OpenCode, Gemini CLI, Cursor Agent, Copilot CLI, Hermes, OpenClaw, and Pi.
  • Supports resume workflows where the underlying Claude command supports resume.
