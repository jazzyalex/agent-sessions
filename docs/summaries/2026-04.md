# 2026-04 Summary

- Cursor sessions now support Resume in Unified Sessions and Copy Resume Command generation with `--resume <chatId>` and `--continue` fallback behavior.
- Analytics indexing is now fully on-demand from the Analytics window. Unified toolbar open no longer triggers automatic builds, and users can start/cancel/update builds with continuous progress visibility.
- Unified/Cockpit responsiveness is prioritized by removing automatic analytics indexing from provider refresh paths when analytics is idle.
- Unified core indexing now runs in a lower-impact foreground profile, slows focused-session live polling, and keeps in-flight refreshes running across app deactivation instead of repeatedly canceling work.
- Unified toolbar indexing affordances now clearly separate core session indexing from analytics indexing so active refresh indicators are less confusing.
- Unified now pauses automatic core monitor refresh loops while the app is active in foreground, then resumes background monitoring when inactive to reduce active-window CPU pressure.
- Unified footer now reports live core indexing progress (`X/Y` sessions and percentage) during active indexing.
- Agent toolbar filters in Unified are now view-only and no longer kick off background re-indexing when toggled.
- Core monitor-triggered refresh work now displays as lightweight syncing status instead of resetting full indexing progress.
- Core restart sync now restores persisted file-stat baselines per source so relaunches no longer re-process large session sets when only a few files changed.
- Codex incremental refresh now detects and reparses recent rollout files that are present on disk but missing from the hydrated snapshot, fixing newest-session dropouts caused by baseline/session mismatch.
- Codex `thread_name` title overrides are now constrained to verified session-root layouts and refreshed with stronger cache invalidation, reducing wrong-title carryover across custom roots and rename edits.
- Cockpit `Go to Session` now allows runtime-ID-backed matches again while still blocking cwd-only fallback matches.
- Session list flat view now shows an `s` badge on subagent sessions so they are identifiable without expanding the hierarchy.
- Session list hierarchy now infers same-workspace parents for older Codex role-only subagent sessions even after multi-hour gaps, reducing stray top-level `s Codex` rows in hierarchical mode.
- Cockpit limits bar auto-expands to show the full detail panel when any quota is amber/red and reset times no longer fit inline.
- Claude Usage now keeps a bounded timed retry active while Cockpit, footer, or menu usage surfaces are visible after OAuth failures, so limits do not wait for Claude terminal prompt activity.
- Custom session titles now supported across Claude (via `/rename` CLI command), Codex (thread_name from session_index.jsonl), Copilot (workspace name from workspace.yaml), and OpenCode (stored session titles from SQLite/JSON, including `/rename`), with proper caching and layout guards per agent.
- Fixed: Claude custom titles from `/rename` were lost on relaunch due to missing column in UPDATE; fixed alongside a gap-scan fix that catches title records in the middle of large session files.
- Resume launch now passes terminal commands to AppleScript via `osascript` argv, reducing command/script interpolation risk.
- Transcript rendering for Whole Session Raw/Pretty now preloads content off the SwiftUI `body` path, and transcript search cache now has bounded memory usage.
