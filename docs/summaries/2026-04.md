# 2026-04 Summary

- Analytics indexing is now fully on-demand from the Analytics window. Unified toolbar open no longer triggers automatic builds, and users can start/cancel/update builds with continuous progress visibility.
- Unified/Cockpit responsiveness is prioritized by removing automatic analytics indexing from provider refresh paths when analytics is idle.
- Unified core indexing now runs in a lower-impact foreground profile, slows focused-session live polling, and keeps in-flight refreshes running across app deactivation instead of repeatedly canceling work.
- Unified toolbar indexing affordances now clearly separate core session indexing from analytics indexing so active refresh indicators are less confusing.
- Unified now pauses automatic core monitor refresh loops while the app is active in foreground, then resumes background monitoring when inactive to reduce active-window CPU pressure.
- Unified footer now reports live core indexing progress (`X/Y` sessions and percentage) during active indexing.
- Agent toolbar filters in Unified are now view-only and no longer kick off background re-indexing when toggled.
- Core monitor-triggered refresh work now displays as lightweight syncing status instead of resetting full indexing progress.
- Core restart sync now restores persisted file-stat baselines per source so relaunches no longer re-process large session sets when only a few files changed.
