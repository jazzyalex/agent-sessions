# 2026-05 Summary

- Analytics time-series chart colors now derive from the actual enabled agents, preventing crashes when Hermes appears in chart data.
- Codex Desktop and Claude Desktop projectless chats now collapse to `Codex Desktop Chats` and `Claude Desktop Chats` so they are easier to filter and identify.
- Analytics project filters now collapse generated worktree, publish-clone, and nested output folder names back to their parent repository.
- Preferences now place About above the agent-specific panes, keeping product details with general app settings.
- Large Pi session files now stay lightweight during ordinary search, and full Pi transcript parsing is size-capped by default unless deep scan explicitly opts in.
- Codex Desktop and Claude Desktop worktree sessions now keep the parent project name, show the worktree as a subtle second line, and avoid selection-driven scroll jumps during hierarchical refreshes.
- Codex Desktop worktree project recovery now uses explicit git origin metadata, so arbitrary worktree names can still display under the parent project without prefix-based guessing.
- Codex state metadata now falls back cleanly for older state database schemas without git origin columns, preserving existing title and cwd enrichment.
- Session list: Added an Archived control that narrows Codex rows to archived Desktop sessions while leaving other enabled agents visible, plus distinct archived styling on the existing `desk` pill.
- Restored Codex Desktop sessions with old rollout dates now sort by fresh activity, are picked up when their files change, and parse non-large unindexed transcripts during search so content such as email addresses is findable without deep scan.
- The menu bar menu now includes Show/Hide Dock Icon and Quit commands, keeps Quit separated, and clears hidden-Dock state when disabling the menu bar item.
- Claude Desktop local-agent transcripts under Claude's Application Support directory are now discovered, enriched with Desktop metadata, and shown in Unified Sessions with the `desk` surface pill.
- Claude rows in Unified Sessions now reuse the same Agent-column `cli` and `desk` surface pills as Codex rows.
- Standard `~/.claude` transcripts launched from Claude Desktop now persist Desktop origin metadata and show the `desk` pill instead of the CLI pill.
- Claude Desktop rows keep the `desk` pill even when cached rows have not yet hydrated generic surface metadata.
- Session origin metadata now has generic originator/source/surface fields so non-Codex providers can use desktop or IDE surface labels without reusing Codex-specific columns.
- Agent format monitoring is verified through Codex 0.130.0, Claude 2.1.140, Gemini 0.41.2, OpenCode 1.14.48, Copilot 1.0.46, and OpenClaw 2026.5.7; the latest checks found no parser, UI, or UX changes needed.
- Pi CLI agent now has tier-2 local support: JSONL session discovery, browsing, search, Preferences controls, Pi colors, Resume/Copy Resume commands via `pi --session`, and weekly session-format monitoring are available; Agent Cockpit/live status, analytics, and usage tracking remain out of scope.
- Warp and WarpPreview are now first-class Resume launch targets
- Terminal picker in Preferences dynamically detects installed terminals via mdfind
